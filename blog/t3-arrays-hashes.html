Containers for Everything You Need

Let me hold that for you

In nearly all real world problems we will be encountering data that is more than a single value. Now is the time to learn how Ruby stores multiple pieces of related information into one vessel. The two most common forms of storage are arrays and hashes. These container type objects share a lot in common, but are useful for different problems. We’ll start by talking about how to create and use both arrays and hashes and conclude by talking about the different uses for each of them.

Creation

  • create blank array: 
    new_array = []
  • create array with data: 
    new_array = [ 23,”favorite number”, [1,1,2,3,5,8] ]
  • create blank hash: 
    new_hash = {}
  • create hash with data: 
    new_hash = { "favorite number" => 23, 
          "address" => 7118, 
          "pets" => ["Harper","Lambeau"] }

Note that any type of object can be stored in an array or a hash (I can put an array in one slot of my array or even as a value of my hash). Keep in mind that there are many other ways of doing the same thing, but for the sake of convenience and simplicity we won’t get into those.

Adding Elements

  • Add an element to an array: 
    my_array << "new value"
  • Add an element to a hash: 
    my_hash["new key"]="new value"

These examples are adding a new value to the end of our collection objects.

Retrieving Elements

  • Retrieve element from an array: 
    a = my_array[3]
  • Retrieve element from a hash: 
    a = my_hash["key"]

Note how the syntax for both of these commands is nearly identical. If we were using a hash and for our key we were using integers and counting by 1 the objects would be nearly identical.

Of course there are many, many other things you can do with collection objects, but these are the most common actions and do a good job of showing how they are both very similar, but different. In addition, they can both take advantage of the enumerable module methods. Both containers can store any type of object (this includes hashes being able to take any object even for a key).

However, there are differences. For one, the order of elements in an array is usually an important aspect. For hashes, the key is meant to be the way of finding elements, not necessarily the order (it is possible, however, to find and use the order that the key/value pairs were entered to take action on hash elements). In Python, there is a very same type of structure that exists, but it is called a dictionary. I think that name is better as it more intuitive to understand the value of that type of container. Hashes are more useful for pairing of data elements name and address, states and abbreviations, city name and population. This container is useful for getting the second piece of related information if I have the first and don’t care what order it was entered. Arrays on the other hand are great for storing lots of data that isn’t necessarily connected with another piece of information. They are spectacular at storing 2-, 3-, and even more dimensions of data. Although hashes have their place, arrays are used much, much more frequently.
