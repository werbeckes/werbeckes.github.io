Blocks, Procs, and Lambdas

A lot better than lions, tigers, and bears

This is the first time I’ve gotten the opportunity to teach you about a concept that I didn’t know existed before Dev Bootcamp. That’s not to say I haven’t learned anything at DBC, I’ve learned more about Ruby than I have about any other coding language I’ve ever worked on. However, the concepts of methods, objects, classes, variables, constants, globals, arrays, hashes, etc. were all something I’ve worked with before when learning Python, Java, Basic, C++, or Cache. I picked this topic because I had literally no idea what it was about and I am excited to explore the world of everything in the coding world that I don’t yet know.

I’ve been using blocks of code for awhile now without knowing what they were called. Below is an example of a block:

array.map! { |x| x+10 }

Everything inside of the curly brackets is a block of code. You can also use the do … end combination to identify the start and end of the block. They work exactly the same as the curly brackets. In this relatively simple example a block works perfectly fine, but in some cases you will want to put significantly more code to pass into a method or even create methods that take multiple blocks of code! To do all of these fancy things we can use Procs or Lambdas.

Below is an example of a Proc.

p = Proc.new{ |x| x+10 }

Essentially, what we are doing is taking the block of code and storing it in a special kind of object called a Proc. Proc is short for Procedure. Ok, now that we have a Proc, what do we do with it? Well, we can use it anywhere we would normally use a block such as:

array.map! { p }

Doesn’t that look simple?! Amazing, right? Taking it a step further, we can create a Lambda using the syntax:

lam = lambda{ |x| x + 10 }

As you can see, the syntax is even simpler than using a Proc. Although, we don’t explicitly create a new object with the Object.new command we’re used to, we are creating a new object. We are actually creating a Proc object! We can then use the lam anywhere we could use a block or a Proc.
