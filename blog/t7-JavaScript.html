de loop

I love looping. I love the concept and I really love the flexibility of Ruby looping. I was actually a little disappointed when i learned about all of the amazing enumerable methods because it meant writing less loops (but of course I came to love Enumerable very quickly). However, the power and variety of Ruby looping meant that I still frequently have to take a quick look back to get the syntax right for different kinds of loops. After learning and using JavaScript loops as well I’ve decided to condense all of the different syntaxes into one blog post for my own sake (as well as yours, my lovely, devoted reader).

While Loop, Initial Check

The while loops for both languages look very similar. The most significant differences are in how to print to the screen after each loop and JS’s need for a semi-colon at the end of each statement. The other difference you see is that JS uses curly brackets to contain the block to be run each loop while the Ruby example is using the do … end to contain the block. However, Ruby allows the use of curly brackets as well.

Ruby:
    count =0
    while count <= 5 do
      p "loop number: #{count}"
      count+=1
    end
JavaScript:
    count = 0;
    while (count <= 5) {
      console.log("loop number:",count);  
      count++;
    }

While Loop, Post Check

Using the check at the end of the loop guarantees that the loop will run at least once. This syntax is very useful for when you want the condition to check user input. Both languages end with a while clause and a condition, but the structure of the block to loop is different. In Ruby the loop is enclosed by begin … end, whereas in JS there is simply a do and curly brackets.

Ruby:
    begin
      p "Give me a number 1-10"
      input = gets.chomp.to_i
    end while (input >= 1 && input <=10)
JavaScript:
    do {
      var input = Number(prompt("Give me a number 1-10"));
    } while (input >= 1 && input <= 10)

Infinite Loop, with Break

Although it’s rarely a good idea to write an infinite loop I put together an example of how one would be written mostly for the sake of not accidentally doing this in the future. In addition, I added how to break the loop from within rather than with a conditional statement. Note that in the Ruby example I also made use of the until command. This is essentially the same as while !, but in certain situations can make your code more reader friendly.

Ruby:
    count = 0
    until false
      p "the loop will never end!"
      if count==1000
        break
      end
      count+=1
    end
JavaScript:
    var count = 0
    while (true) {
      console.log("the loop will never end!");
      count++;
      if (count == 1000) break;
    }

For Loop

And finally, my favorite kind of loop! For loops are used to run a block of code a specific number of times. They are also frequently an option for when you want to loop through an array and perform an action once for each item (however, the enumerable class probably has both an #each and a #map method that are probably better suited for your needs in Ruby). In Ruby, you can very easily say that you want something to happen any number of times with the .times command. With JS your for statement needs to specify three things: setting a variable, the condition to terminate the loop, and the increment after each loop.

Ruby:
    5.times { |i| puts "loop number: #{i}" }
JavaScript:
    for (var count=0; count < 5; count ++) {
      console.log("loop number:",count)
    }

Ruby also has a special kind of for statement when looping through each element of an array:

Ruby:
    for item in array
      puts item
    end
